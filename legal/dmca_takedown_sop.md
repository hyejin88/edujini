# 저작권 침해 신고 대응 SOP

**적용 범위**: EduQA에 등록된 문항·해설·이미지 등 모든 콘텐츠
**대응 책임자**: 정혜진 (단독 운영)
**대응 시간**: 신고 접수 → 24시간 내 1차 대응

---

## 신고 접수 채널

| 채널 | 우선순위 |
|---|---|
| `support@eduqa.app` (가칭) 이메일 | 1순위 |
| 서비스 내 "문제 신고하기" 버튼 | 1순위 |
| 출판사·법무법인 공문 | 1순위 (즉시 대응) |
| SNS DM·댓글 | 2순위 (이메일 안내) |

---

## 24시간 내 1차 대응 절차

### Step 1. 접수 (0~1시간)
- 신고 내용 확인 (어떤 문항 / 어떤 출처 / 어떤 침해 주장)
- 신고자 정보 확인 (개인 / 출판사 / 대리인)
- 접수 번호 부여 (`DMCA-YYYYMMDD-001`)

### Step 2. 임시 비공개 (1~4시간)
- 해당 문항 `is_published = false` 처리
- 사용자 노출 즉시 차단
- 신고자에게 "임시 비공개 처리 완료" 회신

### Step 3. 사실관계 확인 (4~24시간)
- 해당 문항의 `license_code`, 생성 로그, 검증 기록 확인
- 신고자가 제시한 원본과 비교 (실질적 유사성 검토)
- 결과 분류:
  - **명백한 침해** → Step 4-A
  - **회색지대 (패러디·인용 범위)** → Step 4-B
  - **무관** → Step 4-C

### Step 4-A. 명백한 침해
- 영구 삭제
- 신고자에게 사과·삭제 확인 회신
- 동일 출처 다른 문항도 일괄 점검
- 생성 프롬프트 시스템 개선 (재발 방지)

### Step 4-B. 회색지대
- 변호사 30분 자문 (네이버 엑스퍼트 등 약 5만원)
- 자문 결과에 따라 수정·삭제·유지 결정
- 결정 사항을 신고자에게 회신

### Step 4-C. 무관
- 자체 생성 증거(로그·시스템 프롬프트·생성 시각·임의성 입증) 제시
- 신고자에게 정중한 회신
- 필요 시 게시 복구

---

## 문서·증거 보관

- 모든 문항의 **생성 로그** 90일 이상 보관
- 시스템 프롬프트 버전 이력 보관 (Git)
- 분쟁 발생 문항은 **3년 보관** (소송 시효)

---

## 출판사 공문 수신 시 추가 절차

1. 회신 기한 확인 (보통 7~14일)
2. 즉시 임시 비공개
3. 변호사 의뢰 (한국 저작권 전문 변호사, 1차 자문비 약 30만원)
4. 회신 시 다음 사항 명시:
   - 자체 생성 콘텐츠임을 입증할 로그
   - 출처 표시 정책
   - 협력 의향 (라이선스 협상)

---

## 재발 방지 시스템

- 시스템 프롬프트에 `DO NOT copy or paraphrase any actual textbook problem` 명시 ✅ (적용 완료)
- 검증 단계(Pro)에서 "no copyright concerns" 항목 점검 ✅ (적용 완료)
- 분기별 무작위 추출 30문항 → Google 검색 → 5단어 이상 일치 시 경고 알림
- 사용자 신고 누적 3회 이상 출처 → 해당 출판사 스타일 시스템 프롬프트 비활성화
