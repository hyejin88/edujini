# 라이선스 코드 정의 (problems.license_code 컬럼)

| 코드 | 의미 | 출처 예시 | 상업적 이용 | 출처 표시 |
|---|---|---|---|---|
| `SELF_GEN` | EduQA 자체 생성 | Gemini API 기반 자체 제작 | ✅ | EduQA 자체 제작 |
| `GOV_OPEN` | 공공저작물 자유이용 | NCIC 성취기준, 17개 시·도교육청 학력평가 | ✅ | 출처 명시 (예: 2024 서울시교육청 학력평가) |
| `KICE` | 한국교육과정평가원 | 수능, 6·9월 모의평가 | ✅ | "한국교육과정평가원" 명시 |
| `EDU_OFFICE` | 시·도교육청 자체 제작 | 시·도교육청 단위평가, 진단평가 | ✅ | 해당 교육청 명시 |
| `FAIR_USE` | 공정이용 (§35-3) | 일부 인용·평론 목적 | ⚠️ 제한적 | 출처 + 인용 분량 ≤ 10% |
| `THIRD_PARTY` | 제3자 라이선스 (협약 후) | Phase 2: 천재교육 등 출판사 협약 | ✅ | 출판사 표시 + 협약 ID |

## 운영 규칙
1. 모든 problems 레코드는 `license_code` 컬럼 NOT NULL.
2. 게시 전 `scripts/license_validator.py` 통과 필수.
3. `FAIR_USE` 코드는 베타 단계에서 사용 금지 (분쟁 위험).
4. `THIRD_PARTY`는 Phase 2 출판사 영업 성사 후에만 활성화.
5. 분기별 라이선스 분포 리뷰 (`SELF_GEN` 비중 80% 이상 유지).
