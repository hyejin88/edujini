## 2. 핵심 가치 제안 (UVP)

> **"초1부터 고3까지, 모든 교과서의 약점이 어디서 시작됐는지 역추적해 줍니다."**

### 5대 차별 진입 포인트

1. **오답 4축 라벨링** — 개념미숙 / 계산실수 / 문제해석 / 함정미인지
2. **초1~고3 전 교과 단원 그래프 + 선수학습 의존성** (분수 약점이 초3 단원에서 시작됐는지 역추적)
3. **학부모 결제자 ≠ 사용자(학생) 분리 UX** + 주간 카카오 알림톡
4. **B2C → 학원 화이트라벨 SaaS** (한국 학원 7만 개 시장, 좌석당 월 ₩1만 × 500좌석 = ARR ₩60M)
5. **공공누리 + 자체 출제 + AI 변형 문항** (라이선스 리스크 회피)

## 4. 기술 아키텍처

### 4.1 스택

```
[Frontend] Next.js 14 PWA (Tailwind + shadcn/ui + KaTeX)
[Edge]    Cloudflare CDN + Vercel Edge Functions
[Backend] FastAPI (Python) + Celery + Redis Queue
[AI]      GPT-4o/Claude (RAG·해설) + KoBERT (개념 태깅) + IRT 2모수
[Data]    PostgreSQL (Supabase) + pgvector + S3 + BigQuery
[Payment] 토스페이먼츠
```

### 4.2 DB 스키마 8 테이블 (`backend/migrations/001_init.sql`)

`users` / `textbooks` / `units` / `concepts` / `problems` / `problem_concepts` / `attempts` / `diagnosis_reports` (+ 보조 `subscriptions` / `concept_prerequisites`)

**핵심 설계**: NCIC 성취기준 코드 `[4수01-01]` 를 **허브로** 출판사별 단원 매핑 (검정본 5~10종 다양성 흡수). 사용자·개념 임베딩은 `pgvector`(768차원) 사용.

### 4.3 AI 추천 엔진 (3축 결합)

```
약점 점수    × 0.5  (mastery 하위 20% 개념 우선)
난이도 적합도 × 0.3  (현재 θ ± 0.3, 정답률 60~75% 타겟)
간격 반복   × 0.2  (SM-2 변형: 1일/3일/7일/21일)
```

상위 10개 → MMR 다양화 → 5개 추천.
